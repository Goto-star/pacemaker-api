Notion の「PaceMaker｜Task Plan」DB から Order番号 $ARGUMENTS のタスクを取得し、実装から PR 作成まで一貫して行う。

## 手順

### 1. タスク取得
Notion DB「PaceMaker｜Task Plan」を検索し、Order = $ARGUMENTS のタスクを特定する。
以下の項目を読み取る：
- タスク名・概要（なぜやるか）
- 完了条件
- TODO
- やらないこと
- 依存・前提
- AI委任（プロパティ）

### 2. 事前チェック
- **AI委任 = false** なら「これは自分で実装するタスクです」と伝えて終了する
- **依存・前提**に記載されたタスクが未完了なら、その旨を報告して終了する

### 3. ブランチ作成
```
git checkout -b task/$ARGUMENTS-<短い英語スラッグ>
```

### 4. 実装
CLAUDE.md の規約に従い、完了条件をすべて満たす実装とテストを作成する。
- ビジネスロジックは `app/services/` の PORO へ
- コントローラは薄く保つ
- 「やらないこと」に記載された範囲には絶対に手を出さない
- 不明点があれば実装を止めてユーザーに確認する

### 5. 自己チェック
完了条件を1つずつ照合し、以下の観点でテストを見直す：
- DB保存不要なのに `create` を使っていないか
- 1つの `it` に重要検証を詰め込みすぎていないか
- モックで実装詳細を検証していないか（結果・状態・副作用を見ること）
- テスト名だけで仕様が読めるか

### 6. コミット & PR 作成
Conventional Commits 形式でコミットし、PR を作成する。

PR 本文には以下を含める：
- 対象 Order 番号
- 完了条件のチェックリスト（各条件が満たされているか）
- テスト観点

### 7. 報告
作成した PR の番号と URL をユーザーに報告する。
