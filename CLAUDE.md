# CLAUDE.md — PaceMaker API

AIエージェント（Claude / Codex 等）がコードを書く・PRを作る際に従う規約。
PR自動レビューの判断基準にもなる。

---

## 1. プロジェクト概要

独学者向け学習管理アプリのバックエンド。Rails 8 API モード。  
フロント（Next.js）は別リポジトリ。このリポジトリは **API のみ**。  
スケジューリングが核心機能であり、ビジネスロジックはすべてサービス層に集約する。

---

## 2. アーキテクチャの鉄則

- **ビジネスロジックは `app/services/scheduling/` の PORO に置く**
  - `ReviewScheduler` — 復習スケジュール生成
  - `DailyPlanner` — 1日の学習プラン構築
  - `PaceCalculator` — 学習ペース計算
- **コントローラは「受け取る → service を呼ぶ → 返す」だけ。fat controller 禁止**
- **モデルは薄く保つ**。バリデーション・アソシエーション・スコープのみ。ロジックを書かない
- サービス間の依存は引数で渡す。グローバルステートに頼らない

---

## 3. コーディング規約

- Rails 8 / Ruby 3.3 の作法に従う
- Lint は **RuboCop rails-omakase**（`bundle exec rubocop`）
- DB 変更は必ずマイグレーションを作成する。`schema.rb` を手編集しない
- 環境変数は `ENV.fetch("KEY")` で取得し、デフォルト値が必要な場合のみ第2引数を指定する
- ルーティングは resources を使い、カスタムアクションは最小限にする

---

## 4. テスト方針（RSpec）

- `describe` はテスト対象、`context` は条件、`it` は期待結果を表す。3つを繋げると仕様の文になること
- 1つの `it` に重要検証を詰め込みすぎない
- DB 保存が不要なら `create` を使わない。まず `build` / `build_stubbed` を検討する
- `let` を増やしすぎない。スコープが広がるほど読みにくくなる
- モックで実装詳細を検証しない。出力・状態・副作用の **結果** を検証する
- テスト名だけで仕様がわかること
- リファクタで壊れにくい設計にする（内部実装ではなく振る舞いをテスト）
- テストは実装と **同じ PR** に含める。後回し禁止

---

## 5. 認証・秘密情報

- `.env`・API キー・シークレットをコードや PR diff に含めない
- 秘密情報は Rails credentials または環境変数で管理する
- `config/credentials.yml.enc` 以外に平文のキーを置かない

---

## 6. PR 作法

| 項目 | ルール |
|------|--------|
| 対応範囲 | 1チケット = 1PR |
| ブランチ名 | `task/<Order>-<英語スラッグ>`（例: `task/42-review-scheduler`）|
| コミットメッセージ | Conventional Commits（`feat:` `fix:` `chore:` `test:` `refactor:` 等）|
| PR タイトル | Conventional Commits 形式で日本語 OK |
| PR description | 概要・変更内容・テスト観点を記載する |

---

## 7. やってはいけないこと

- チケットの「やらないこと」に記載された項目を実装する
- スコープ外の機能・リファクタを勝手に追加する
- 不明点があるのに推測で進める → **止めて確認する**
- `schema.rb` を直接編集する
- テストなしで PR を出す
- 秘密情報をコミットする

---

## 8. タスクの探し方

Notion DB **「PaceMaker｜Task Plan」** を参照する。  
ブランチ名・PR タイトルの `<Order>` は Notion の Order 番号と対応させる。  
実装前にチケットの「やること／やらないこと」を必ず確認する。

---

## 9. PaceMaker 固有ルール

一般的な間隔反復（SM-2）の知識と異なる点があるため必ず確認すること。

- **定着度** のラベルは「定着度」で統一する。「節約率」と書かない
- **理解度評価は ★1〜3 の3段階**（SM-2 標準の 0〜5 quality ではない）
  - ★3 → 復習間隔を大きく伸ばす
  - ★1 → 復習間隔をリセット
- **優先度は固定順**：復習 → 締切間近の新規 → 通常の新規
- 定着度は節約率の近似値として扱う（厳密な計測はしない）
